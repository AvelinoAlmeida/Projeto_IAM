\section{Justificação da Escolha do Keycloak}

O Keycloak \cite{KeycloakDocs} é um \gls{idp} de código aberto desenvolvido e mantido pela Red Hat, que implementa os protocolos \gls{oidc}, \gls{oauth2} e SAML~2.0. A versão~26 (utilizada neste projeto) introduz melhorias de desempenho e suporte a \textit{passkeys}.

As razões que justificam a sua escolha para este trabalho são:

\begin{enumerate}
  \item \textbf{Custo zero} — sem licenciamento, adequado para um projeto académico;
  \item \textbf{Auto-hospedado via Docker} — permite controlo total e execução local reproduzível;
  \item \textbf{Exportação do realm} — \texttt{realm-export.json} versionado em git garante configuração declarativa;
  \item \textbf{Suporte nativo a \gls{mfa} \gls{totp}} — configurável por utilizador via \textit{Required Actions};
  \item \textbf{Admin \gls{rest} \gls{api} completa} — permite automação do ciclo de vida \gls{jml} de forma programática;
  \item \textbf{Revogação de sessões} — suporte a \texttt{DELETE /users/\{id\}/sessions} para efeito imediato;
  \item \textbf{Auditoria integrada} — registo de eventos de utilizador e de administração no PostgreSQL.
\end{enumerate}

\section{Abordagens de Validação de Tokens}

Existem duas estratégias principais para validar \gls{jwt} num \textit{resource server}:

\begin{itemize}
  \item \textbf{Token Introspection} (RFC~7662): o \textit{resource server} chama o \gls{idp} em cada pedido para verificar a validade do token. Garante revogação imediata, mas introduz latência e dependência de disponibilidade;
  \item \textbf{Validação local via JWKS}: o \textit{resource server} obtém as chaves públicas do \gls{idp} uma vez (e as cacheia) e verifica a assinatura criptográfica localmente. Elimina a latência extra, mas não deteta revogação até à expiração do token.
\end{itemize}

Neste projeto, adota-se a \textbf{validação local via \gls{jwks}} (\textit{cache} em memória), compensando a limitação de revogação com a estratégia de revogação explícita de sessões nos fluxos \gls{jml} (Mover e Leaver). Esta combinação oferece o melhor equilíbrio entre desempenho e segurança para o cenário implementado.
