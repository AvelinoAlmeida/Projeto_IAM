O que é IAM?
🔐

Autenticação

Quem és tu? OIDC via Keycloak. JWT assinado RS256. Fluxo Authorization Code + PKCE.

🛡️

Autorização (RBAC)

O que podes fazer? 3 roles: admin, colaborador, visitante. Endpoints protegidos por role.

🔒

MFA

Segunda camada de segurança. TOTP via Google Authenticator. Obrigatório para admin.

👥

Ciclo de Vida (JML)

Joiner, Mover, Leaver. Revogação de sessões em tempo real via Keycloak Admin API.

Fases do Projeto
1

Infraestrutura Base

Docker Compose com 3 serviços. realm-export.json versionado. Arranque reprodutível em 2–3 min.

2

Autenticação OIDC

JWT RS256 emitido pelo Keycloak. Validação de assinatura via JWKS. Claims: sub, email, roles.

3

Autorização RBAC

3 roles: admin, colaborador, visitante. Endpoints com respostas 200/403 demonstráveis ao vivo.

4

Autenticação MFA

TOTP via Google Authenticator. admin.user com CONFIGURE_TOTP. Verificação via claim amr/acr.

5

Ciclo de Vida (JML)

Joiner · Mover · Leaver via Keycloak Admin API. Sessões revogadas imediatamente na mudança de role.

6

Auditoria

Eventos Keycloak: LOGIN, LOGOUT, LOGIN_ERROR. Dashboard com filtro e resumo por tipo de evento.

Setup MFA — admin.user

      

        

          1
          Abrir http://localhost:8081 → Realm iam-tp
        

        

          2
          Login como admin.user / Admin@1234 → o Keycloak pede setup TOTP
        

        

          3
          Scan QR code com Google Authenticator ou Authy
        

        

          4
          Inserir o OTP de 6 dígitos → TOTP configurado ✅
        

        

          5
          Próximos logins pedem: password + OTP (browser-mfa flow)
        

      

      

        ✅ admin.user tem CONFIGURE_TOTP como Required Action no realm-export.json
